Python Tupel

Einführung in Tupel

Tupel sind eine grundlegende Datenstruktur in Python, die ähnlich wie Listen verwendet wird, aber im Gegensatz zu Listen unveränderbar ist. Einmal erstellt, können die Elemente eines Tupels nicht mehr verändert werden.

Weitere Informationen zu Tupeln in Python finden Sie in der offiziellen Python-Dokumentation.

Erstellen von Tupeln

Tupel können erstellt werden, indem man die Elemente in runde Klammern () schreibt, getrennt durch Kommas. Tupel können auch ohne Klammern erstellt werden, wenn mehrere Werte durch Kommas getrennt sind.

# Ein einfaches Tupel
person = ("Alice", 30, "Berlin")
print(person)

# Tupel ohne Klammern
numbers = 1, 2, 3
print(numbers)

Zugriff auf Werte in Tupeln

Auf Werte in einem Tupel kann mit Hilfe ihrer Indizes zugegriffen werden. Die Indizierung beginnt bei 0.

# Zugriff auf Werte über Index
person = ("Alice", 30, "Berlin")
name = person[0]
alter = person[1]
stadt = person[2]
print(name, alter, stadt)

Entpacken von Tupeln

Sie können die Elemente eines Tupels in einzelne Variablen entpacken. Dies wird auch Tuple Unpacking genannt.

# Entpacken eines Tupels
person = ("Alice", 30, "Berlin")
name, alter, stadt = person
print(name)
print(alter)
print(stadt)

Tupel sind unveränderbar

Einmal erstellt, können die Elemente eines Tupels nicht mehr verändert werden. Das bedeutet, dass Sie weder Elemente hinzufügen noch entfernen oder ändern können.

# Unveränderbarkeit von Tupeln
person = ("Alice", 30, "Berlin")
# Dies führt zu einem Fehler:
# person[1] = 31

Verschachtelte Tupel

Tupel können andere Tupel als Elemente enthalten. Dies ermöglicht die Erstellung komplexer Datenstrukturen.

# Verschachtelte Tupel
nested_tupel = (1, (2, 3), (4, 5, 6))
print(nested_tupel)

# Zugriff auf verschachtelte Elemente
inner_tupel = nested_tupel[1]
print(inner_tupel)
print(inner_tupel[1])

Tupel mit einer Liste vergleichen

Obwohl Tupel und Listen ähnlich aussehen, gibt es einige wichtige Unterschiede zwischen ihnen. Der Hauptunterschied besteht darin, dass Tupel unveränderbar sind, während Listen veränderbar sind.

# Vergleich zwischen Tupeln und Listen
tupel = (1, 2, 3)
liste = [1, 2, 3]

# Tupel sind unveränderbar
try:
    tupel[0] = 0
except TypeError as e:
    print(e)

# Listen sind veränderbar
liste[0] = 0
print(liste)
