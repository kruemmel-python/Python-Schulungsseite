Python Mocking

Einführung in Mocking

Mocking ist eine Technik im Unit-Testing, bei der Objekte oder Verhaltensweisen simuliert werden, um isolierte Tests zu ermöglichen. Weitere Informationen zum Mocking in Python finden Sie in der unittest.mock-Dokumentation.

Grundlagen des Mockings

Python bietet mit dem unittest.mock-Modul eine integrierte Bibliothek zum Erstellen und Verwenden von Mock-Objekten.

Beispiele für Mocking

Hier sind einige Beispiele für die Verwendung von Mocking in Python:

# Einfaches Mocking mit unittest.mock
from unittest.mock import Mock

# Ein Mock-Objekt erstellen
mein_mock = Mock()

# Methodenaufrufe simulieren
mein_mock.eine_methode.return_value = 3

# Verwenden des Mock-Objekts
ergebnis = mein_mock.eine_methode(1, 2)
print(ergebnis)  # Ausgabe: 3

Mocking in Unit-Tests

Mocking kann in Unit-Tests verwendet werden, um Abhängigkeiten zu simulieren und zu isolieren.

# Mocking in Unit-Tests
import unittest
from unittest.mock import patch

def addiere(a, b):
    return a + b

class TestAddiere(unittest.TestCase):
    @patch('__main__.addiere', return_value=5)
    def test_addiere(self, mock_addiere):
        ergebnis = addiere(1, 2)
        self.assertEqual(ergebnis, 5)

if __name__ == '__main__':
    unittest.main()

Weitere Mocking-Techniken

Es gibt viele weitere Techniken und Methoden, die mit unittest.mock verwendet werden können, wie das Mocken von Attributen und das Überprüfen von Methodenaufrufen.

# Mocking von Attributen
class Person:
    def __init__(self, name):
        self.name = name

person_mock = Mock()
person_mock.name = "Alice"
print(person_mock.name)  # Ausgabe: Alice

# Überprüfen von Methodenaufrufen
mein_mock = Mock()
mein_mock.eine_methode(1, 2)
mein_mock.eine_methode.assert_called_with(1, 2)

Zusammenfassung

Mocking ist ein wesentlicher Bestandteil des Unit-Testings, der es ermöglicht, isolierte und zuverlässige Tests zu schreiben. Weitere Informationen finden Sie in der unittest.mock-Dokumentation.
