Python Unit-Testing

Einführung in Unit-Testing

Unit-Testing ist eine Technik in der Softwareentwicklung, bei der einzelne Einheiten oder Komponenten einer Software getestet werden, um sicherzustellen, dass sie wie erwartet funktionieren. Weitere Informationen zum Unit-Testing in Python finden Sie in der unittest-Dokumentation.

Grundlagen des Unit-Testings

Python bietet mit dem unittest-Modul eine integrierte Bibliothek zum Erstellen und Ausführen von Unit-Tests. Alternativ gibt es auch andere beliebte Test-Frameworks wie pytest.

Beispiele für Unit-Testing

Hier sind einige Beispiele für die Verwendung von Unit-Testing in Python:

# Einfache Unit-Tests mit unittest
import unittest

def addiere(a, b):
    return a + b

class TestAddiere(unittest.TestCase):
    def test_addiere(self):
        self.assertEqual(addiere(1, 2), 3)
        self.assertEqual(addiere(-1, 1), 0)
        self.assertEqual(addiere(-1, -1), -2)

if __name__ == '__main__':
    unittest.main()

Unit-Testing mit pytest

pytest ist ein weiteres beliebtes Framework für Unit-Tests in Python. Es ist einfacher zu verwenden und bietet mehr Funktionen als unittest.

# Einfache Unit-Tests mit pytest
def addiere(a, b):
    return a + b

def test_addiere():
    assert addiere(1, 2) == 3
    assert addiere(-1, 1) == 0
    assert addiere(-1, -1) == -2

Weitere Beispiele und Best Practices

Es gibt viele Best Practices und erweiterte Techniken für das Unit-Testing in Python, wie das Testen von Ausnahmebehandlungen, das Verwenden von Fixtures und das Mocking von Objekten.

# Testen von Ausnahmebehandlungen
def divide(a, b):
    if b == 0:
        raise ValueError("Division durch Null")
    return a / b

class TestDivide(unittest.TestCase):
    def test_divide(self):
        self.assertEqual(divide(10, 2), 5)
        with self.assertRaises(ValueError):
            divide(10, 0)

# Verwenden von pytest Fixtures
import pytest

@pytest.fixture
def zahl():
    return 42

def test_zahl(zahl):
    assert zahl == 42

# Mocking von Objekten
from unittest.mock import MagicMock

def test_mock():
    mock = MagicMock()
    mock.return_value = 10
    assert mock() == 10

Zusammenfassung

Unit-Testing ist ein wesentlicher Bestandteil der Softwareentwicklung, der sicherstellt, dass einzelne Komponenten korrekt funktionieren. Es gibt verschiedene Werkzeuge und Techniken, um effektive Unit-Tests in Python zu schreiben. Weitere Informationen finden Sie in der unittest-Dokumentation und der pytest-Dokumentation.
